# Аудит дизайна (top-design + responsive-design)

## Оценка по рубрике top-design

| Категория   | Вес  | Оценка | Комментарий |
|------------|------|--------|-------------|
| Типографика | 25% | 7/10  | Fluid scale (hero/display/title/body), Geist variable, tracking -0.02em, text-balance. Иерархия ясная. |
| Композиция  | 25% | 8/10  | Асимметричный hero (8.33%), блок «Главная тема», сетка новостей. «Активные обсуждения» — реальные темы и ссылки. |
| Движение    | 20% | 7/10  | Custom easing (expo/quart), hover scale на карточках и кнопках, появление hero по очереди. Скролл по умолчанию. |
| Цвет        | 15% | 8/10  | Тёплые #fafaf9/#0c0c0c, один акцент (accent-signature), ::selection. |
| Детали      | 15% | 7.5/10| min-h 44px, focus-visible, skip link, reduced-motion, CTA в шапке с акцентом. |

**Итого: ~7.5/10** — уровень сильного продукта, выше типичных форумов.

## Что уже хорошо (vs конкуренты)

- Нет чистого чёрного/белого; тёплая палитра и один акцент.
- Fluid-типографика и контраст hero/body.
- Асимметрия hero и смещение влево (lg:ml-[8.33%]).
- Единые кривые анимации (cubic-bezier), не linear/ease.
- Крупные зоны нажатия (44px), доступность (skip, focus, lang, reduced-motion).
- Карточки новостей и «Главная тема» с hover и градиентом.
- Реальный контент в статьях и в блоке «Активные обсуждения».

## Что можно усилить дальше

- **Плавный скролл**: Lenis или Locomotive Scroll для ощущения «премиум».
- **Типографика hero**: при желании — более выразительный display-шрифт для заголовка.
- **Signature moment**: один сильный визуальный момент (например, scroll-reveal или крупный тип).
- **Контейнерные запросы**: для карточек в сетке (responsive-design), если нужна адаптация по ширине контейнера.

## Quick diagnostic (top-design)

| Вопрос | Статус |
|--------|--------|
| Hero заставляет остановиться при скролле? | Да, размер и контраст |
| Есть ли «скриншотный» момент? | Да, блок «Главная тема» |
| Иерархия читается при размытии? | Да |
| Все easing кастомные? | Да |
| Есть асимметрия? | Да |
| Цвета «свои» под проект? | Да |
| Появление при загрузке по очереди? | Да (hero-stagger) |
| Микро-детали (selection, focus)? | Да |
